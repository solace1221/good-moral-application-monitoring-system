# 8-Minute Demo Guide

Use this guide for a timed live demo of GMAMS. Practice until each section fits within its time window.

---

## Timed Script

### 0:00 — Introduction (1 minute)

> "This is the Good Moral Application and Monitoring System, or GMAMS, developed for Saint Paul University Philippines. It replaces the manual paper-based process for requesting and issuing Good Moral Certificates."

- Show the landing / login page.
- Mention the 7 user roles.

---

### 1:00 — Student: Apply for Good Moral Certificate (1.5 minutes)

1. Log in as **student**.
2. Navigate to **Applications → New Application**.
3. Fill in the request form: reason, supporting document.
4. Submit the application.
5. Show the **Application Status** page — status shows "Pending".

---

### 2:30 — Admin: Review and Approve (1.5 minutes)

1. Log out. Log in as **admin**.
2. Navigate to **Applications → Pending**.
3. Open the student's application.
4. Click **Approve**.
5. Show the status change to "Approved".

---

### 4:00 — Admin: Generate Certificate (2 minutes)

1. Navigate to **Certificates**.
2. Find the approved application.
3. Click **Generate Certificate**.
4. Show the generated PDF certificate with:
   - Student name and course
   - Academic year and semester
   - College population figure
   - Dean signature line

---

### 6:00 — SecOSA: Violation Tracking (1 minute)

1. Log out. Log in as **SecOSA**.
2. Navigate to **Violations**.
3. Show an existing student's violation record (major and minor counts).
4. Explain: applications are automatically blocked when a student has active violations.

---

### 7:00 — Dean Dashboard (30 seconds)

1. Log out. Log in as **Dean**.
2. Show the Dean dashboard: pending applications for the college, violation summaries.

---

### 7:30 — Closing (30 seconds)

> "GMAMS digitizes the entire workflow — from student request to certificate issuance — with complete audit trail, role-based access, and CMS integration for enrollment validation."

---

## Tips

- Pre-create a pending application before the live demo to save time.
- Have all role accounts logged out and credentials ready.
- Use two browser windows or profiles to switch roles faster.
